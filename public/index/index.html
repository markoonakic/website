Homelab Remote Access Architecture

This post explains how I remotely access my Kubernetes homelab cluster through a secure architecture using Traefik, MetalLB, and cert-manager. Architecture Overview Traffic Flow All external traffic follows the same path through the ingress layer: Internet → Router: All requests (HTTP/HTTPS on 80/443 and Headscale DERP on 3478) arrive at the home router Router → MetalLB: Port forwarding sends all traffic to the MetalLB virtual IP (192.168.0.220) MetalLB → Traefik: MetalLB load balancer distributes traffic to Traefik service across both cluster nodes
